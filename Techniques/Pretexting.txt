1️⃣ Information

সংজ্ঞা:
Pretexting হলো সোশ্যাল ইঞ্জিনিয়ারিং আক্রমণের একটি ধরণ, যেখানে আক্রমণকারী মিথ্যা পরিচয় বা গল্প তৈরি করে ভিকটিমকে ব্যক্তিগত তথ্য, credential বা সংবেদনশীল তথ্য শেয়ার করতে প্ররোচিত করে।

বৈশিষ্ট্য:

ভুয়া পরিচয় বা কাহিনী তৈরি

ডেলিভারি চ্যানেল: ফোন কল, ইমেইল, SMS, সোশ্যাল মিডিয়া

উদ্দেশ্য: Credential theft, sensitive data exfiltration

গবেষণার জন্য ব্যবহার: সিমুলেশন, ট্রেনিং, এডুকেশনাল প্র্যাকটিস


ধাপসমূহ (Attack Lifecycle – গবেষণার দৃষ্টিকোণ)

1. Reconnaissance – ভিকটিম সম্পর্কিত তথ্য সংগ্রহ (OSINT, সোশ্যাল মিডিয়া, পাবলিক রেকর্ড)


2. Pretext Creation – বিশ্বাসযোগ্য ভুয়া পরিচয় বা গল্প তৈরি


3. Delivery – ফোন কল, ইমেইল, বা SMS পাঠানো


4. Exploitation – ভিকটিমকে তথ্য শেয়ার করতে প্ররোচিত করা


5. Actions on Objectives – ডেটা লগিং, interaction tracking, response analysis



টেকনিক:

Identity spoofing (মিথ্যা পরিচয়)

Urgency বা Threat creation

Authority impersonation (ব্যাংক, অফিস, সরকারী কর্মকর্তা)

Scripted interaction for persuasion

Cross-channel pretexting (email + phone)

2️⃣ Method

Lab Setup:

VirtualBox / VMware – isolated lab VM

Kali Linux / Ubuntu VM

Android Emulator (Android Studio / Genymotion)

Isolated network / sandbox environment


Pretexting Simulation:

Create scripted scenarios for phone calls, emails, SMS

Simulated identities for lab interactions

Log interactions (responses, data submission, compliance)

Measure success rate in lab environment


Data Collection & Analysis:

Interaction rate, compliance rate

Statistical metrics: precision, recall, F1-score

Visualization: interaction timeline, heatmaps


Ethical / Legal Considerations:

All target data simulated

Licensed/open-source tools only

Sandbox / network isolation
