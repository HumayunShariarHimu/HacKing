1️⃣ Information

সংজ্ঞা:
Session Fixation হলো ওয়েব অ্যাপ্লিকেশন আক্রমণের একটি ধরণ যেখানে আক্রমণকারী ভিকটিমকে একটি নির্দিষ্ট session ID ব্যবহার করতে প্ররোচিত করে, যাতে ভিকটিম লগইন করার পর আক্রমণকারী সেই session ID দিয়ে unauthorized access পেতে পারে।

বৈশিষ্ট্য:

লক্ষ্য: ওয়েব অ্যাপ্লিকেশন, SaaS, ক্লাউড সেশন

Credential চুরি না করেও session control

উদ্দেশ্য: Unauthorized access, data theft, privilege escalation

গবেষণার জন্য ব্যবহার: সিমুলেশন, penetration testing, security training


ধাপসমূহ (Attack Lifecycle – গবেষণার দৃষ্টিকোণ)

1. Reconnaissance – টার্গেট ওয়েব অ্যাপ্লিকেশন ও session management পর্যবেক্ষণ


2. Session ID Setup – lab-simulated fixed session ID তৈরি


3. Victim Interaction – ল্যাব environment এ ভিকটিমকে fixed session ID ব্যবহার করতে প্ররোচিত করা


4. Exploitation – ভিকটিম লগইন করার পর session ID ব্যবহার করে lab-only unauthorized access


5. Actions on Objectives – data access, session replay, privilege simulation



টেকনিক:

Predefined session ID injection

URL / cookie manipulation

Session fixation via GET/POST parameters

Cross-Site Scripting (XSS) / phishing simulation for session fixation

Lab-only token replay & access logging

Kali Linux / Ubuntu VM

Web application sandbox (DVWA, WebGoat)

Android Emulator (Android Studio / Genymotion)

Isolated network for lab simulation


Session Fixation Simulation:

Create fixed session ID in lab web application

Simulate victim login with pre-set session ID

Capture lab access using that session ID

Log actions and unauthorized access attempts


Data Collection & Analysis:

Fixed session usage metrics

Unauthorized access success rate

Statistical analysis: login attempts, session hijack success

Visualization: session timeline, access logs


Ethical / Legal Considerations:

All web applications & users are lab-simulated
