Directory Traversal Attack (ডিরেক্টরি ট্র্যাভার্সাল আক্রমণ)

Directory Traversal Attack (যাকে Path Traversal Attack বা "Dot-Dot-Slash Attack"ও বলা হয়) একটি সিকিউরিটি দুর্বলতা যা একটি ওয়েব অ্যাপ্লিকেশন বা সিস্টেমে ঘটে, যেখানে আক্রমণকারী সিস্টেমের ফাইল সিস্টেমের বাহিরে থাকা সংবেদনশীল বা নিরাপত্তাহীন ফাইলগুলোতে অ্যাক্সেস পেতে পারে। এই আক্রমণটির মাধ্যমে আক্রমণকারী তার ইনপুটের মাধ্যমে ফাইল সিস্টেমের অন্যান্য ডিরেক্টরিতে পৌঁছাতে পারে এবং সেখান থেকে গুরুত্বপূর্ণ ফাইল যেমন কনফিগারেশন ফাইল, পাসওয়ার্ড ফাইল ইত্যাদি পেতে পারে।

এই আক্রমণটি সাধারণত ওয়েব অ্যাপ্লিকেশনগুলিতে ঘটে যেখানে ব্যবহারকারীর ইনপুটকে ফাইল পাথ হিসেবে গ্রহণ করা হয়, এবং ইনপুটটি সঠিকভাবে যাচাই করা হয় না।

কিভাবে Directory Traversal Attack কাজ করে?

Directory Traversal Attack সাধারণত একটি URL প্যারামিটার বা ফাইল ইনপুট মাধ্যমে শুরু হয়। আক্রমণকারী ফাইল পাথের অংশ হিসেবে ../ (dot-dot-slash) ব্যবহার করে সিস্টেমের ফাইল সিস্টেমের অন্য ডিরেক্টরিতে প্রবেশ করতে পারে।

উদাহরণ:

সাধারণ URL: http://example.com/file.php?file=profile.txt

আক্রমণের জন্য URL: http://example.com/file.php?file=../../../../etc/passwd


এখানে ../ পাথের এক স্তরের উপরে চলে যাওয়ার নির্দেশনা দেয়, এবং আক্রমণকারী এর মাধ্যমে গুরুত্বপূর্ণ ফাইল যেমন /etc/passwd বা অন্যান্য সিস্টেম ফাইল অ্যাক্সেস করতে পারে।

কিভাবে Directory Traversal Attack হয়?

1. ফাইল ইনপুট: যদি একটি ওয়েব অ্যাপ্লিকেশন বা সার্ভার ব্যবহারকারীর ইনপুটকে ফাইল পাথ হিসেবে গ্রহণ করে, এবং সঠিকভাবে ইনপুটের যাচাই না করে, তবে আক্রমণকারী এই ইনপুট ব্যবহার করে ../ ব্যবহার করে ফাইল সিস্টেমে অন্যান্য ডিরেক্টরি অ্যাক্সেস করতে পারে।


2. ইনপুট বৈধতা যাচাইয়ের অভাব: যদি ইনপুটের বৈধতা যাচাই করার সময় যথাযথ সুরক্ষা ব্যবস্থা না থাকে, আক্রমণকারী ফাইল পাথের অংশ হিসেবে অযাচিত ডিরেক্টরির অ্যাক্সেস চেয়ে এই আক্রমণ চালাতে পারে।


3. ফাইল সিস্টেমের সঠিক কনফিগারেশন না থাকা: ফাইল সিস্টেম বা সার্ভারের সঠিক কনফিগারেশন এবং নিরাপত্তা পলিসি না থাকার কারণে আক্রমণকারীরা /etc/, /bin/ বা অন্যান্য সিস্টেম ফোল্ডার অ্যাক্সেস করতে পারে।



Directory Traversal Attack এর উদাহরণ:

ধরা যাক একটি ওয়েব অ্যাপ্লিকেশন এমন একটি ফিচার প্রদান করছে যেখানে ব্যবহারকারীরা ফাইল পাথ ইনপুট দিতে পারে। উদাহরণস্বরূপ:

http://example.com/viewfile.php?file=images/profile.jpg

এতে ফাইল পাথ ইনপুটে images/profile.jpg ব্যবহারকারী দেখবে। কিন্তু, যদি আক্রমণকারী URL এর মধ্যে ../ ব্যবহার করে:

http://example.com/viewfile.php?file=../../../../etc/passwd

তাহলে এটি সিস্টেমের /etc/passwd ফাইলটি দেখানোর চেষ্টা করবে, যা একটি নিরাপত্তাহীন ফাইল এবং ব্যবহারকারীর পাসওয়ার্ড সম্পর্কিত তথ্য ধারণ করে।

Directory Traversal Attack এর প্রভাব:

1. সংবেদনশীল ফাইল অ্যাক্সেস: আক্রমণকারী সিস্টেমের গুরুত্বপূর্ণ ফাইল অ্যাক্সেস করতে পারে, যেমন কনফিগারেশন ফাইল, পাসওয়ার্ড ফাইল, ডাটাবেস ফাইল, এবং সিস্টেম লগ ফাইল।


2. সিস্টেম কনফিগারেশন চুরি: আক্রমণকারী সিস্টেমের কনফিগারেশন ফাইল চুরি করতে পারে, যা পরবর্তীতে আক্রমণ বা তথ্য চুরির জন্য ব্যবহার করা যেতে পারে।


3. অবৈধ ফাইল এক্সিকিউশন: আক্রমণকারী সিস্টেমের এমন ফাইল অ্যাক্সেস করতে পারে যা এক্সিকিউটেবল, এবং সেগুলি রান করে সিস্টেমে কোড এক্সিকিউট করতে পারে।



কিভাবে Directory Traversal Attack প্রতিরোধ করা যায়?

1. ইনপুট ভ্যালিডেশন: সমস্ত ইনপুট, বিশেষত ইউআরএল প্যারামিটার এবং ফাইল পাথকে যাচাই করা উচিত। ../ এর মতো পাথ ট্র্যাভার্সাল সিকোয়েন্স ব্লক করতে হবে।


2. এনকোডিং: ব্যবহারকারীর ইনপুটে বিশেষ চিহ্ন বা সিকোয়েন্সের জন্য এনকোডিং ব্যবহার করা উচিত। উদাহরণস্বরূপ, ../ কে এনকোড করে %2e%2e%2f এরূপ ব্যবহার করা।


3. সীমিত ডিরেক্টরি অ্যাক্সেস: ওয়েব অ্যাপ্লিকেশনকে কেবলমাত্র নির্দিষ্ট ফোল্ডারে প্রবেশের অনুমতি দেয়া উচিত এবং বাইরের ডিরেক্টরি অ্যাক্সেস নিষিদ্ধ করা উচিত। শুধুমাত্র নির্দিষ্ট ডিরেক্টরি পাথের মধ্যে ফাইল অ্যাক্সেস দেওয়া উচিত।


4. ফাইল পাথের নিরাপত্তা: ফাইল পাথ সিস্টেমের বাইরে যাওয়ার অনুমতি দেয় এমন কোন স্ক্রিপ্ট বা কোড থাকা উচিত নয়।


5. ফাইল সিস্টেম কনফিগারেশন: সিস্টেমের কনফিগারেশন যথাযথভাবে নিরাপত্তা নির্দেশাবলী মেনে চলা উচিত। এর মাধ্যমে সিস্টেম ফাইলগুলোর নিরাপত্তা সুরক্ষা বাড়ানো যাবে।



উপসংহার:

Directory Traversal Attack একটি সাধারণ, কিন্তু গুরুত্বপূর্ণ সিকিউরিটি দুর্বলতা যা ওয়েব অ্যাপ্লিকেশনগুলিতে ঘটে। এটি প্রতিরোধ করতে ইনপুট ভ্যালিডেশন, নিরাপদ কনফিগারেশন এবং ফাইল সিস্টেমের অ্যাক্সেস সুরক্ষিত করা জরুরি।
