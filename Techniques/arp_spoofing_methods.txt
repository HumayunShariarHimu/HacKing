ARP Spoofing (ARP Poisoning) আক্রমণ হলো একটি নেটওয়ার্কে ARP (Address Resolution Protocol) তথ্য ম্যানিপুলেট করে এক ডিভাইসের MAC অ্যাড্রেসকে অন্য ডিভাইসের MAC অ্যাড্রেস হিসেবে প্রতিস্থাপন করা, যাতে আক্রমণকারী টার্গেট ডিভাইস থেকে আসা নেটওয়ার্ক ট্রাফিক ফাঁস বা ম্যানিপুলেট করতে পারে।

এটি Man-in-the-Middle (MITM) আক্রমণের অংশ হতে পারে, যেখানে আক্রমণকারী নেটওয়ার্কের মধ্যে দুইটি ডিভাইসের মধ্যে সন্নিবেশিত থেকে তাদের ট্রাফিক মনিটর বা পরিবর্তন করতে পারে।

ARP Spoofing করার কৌশল ও টেকনিক:

1. ARP Request & ARP Reply মেকিং:

ARP Spoofing আক্রমণকারী ARP Request এবং ARP Reply প্রেরণ করে। আক্রমণকারী অন্য ডিভাইসকে ভুয়া MAC অ্যাড্রেস পাঠায়। এতে গন্তব্য ডিভাইস তার ARP টেবিল আপডেট করে এবং আক্রমণকারীর MAC অ্যাড্রেস সঠিক IP অ্যাড্রেস হিসেবে গ্রহণ করে।

ARP Request: একটি ডিভাইস যখন অন্য ডিভাইসের MAC অ্যাড্রেস জানতে চায়, তখন এটি ARP Request পাঠায়।

ARP Reply: ARP Request-এর প্রতিক্রিয়া হিসেবে সঠিক ডিভাইস MAC অ্যাড্রেস পাঠায়।


ARP Spoofing করার জন্য আক্রমণকারী ARP Reply পাঠায় যেখানে IP অ্যাড্রেস সঠিক কিন্তু MAC অ্যাড্রেস আক্রমণকারীর অ্যাড্রেস হবে।

2. Network Sniffing & Injection:

আক্রমণকারী নেটওয়ার্কের মধ্যে থাকা সমস্ত ARP Request এবং ARP Reply শুনে নেয় এবং তারপরে ARP Poisoning করার জন্য প্রয়োজনীয় ম্যানিপুলেশন করে। এটি ARP টেবিলের ডাটা ভুয়া করে অন্য ডিভাইসে পাঠানো হয়।

Packet Injection: আক্রমণকারী নেটওয়ার্কে প্যাকেট ইনজেক্ট করে ARP টেবিল আপডেট করে। এটি ট্রাফিক রিডাইরেক্ট করতে সাহায্য করে এবং আক্রমণকারী টার্গেটের ট্রাফিক দেখতে বা ম্যানিপুলেট করতে সক্ষম হয়।


3. Man-in-the-Middle (MITM) Attack:

ARP Spoofing-এর মাধ্যমে আক্রমণকারী MITM Attack চালাতে পারে, যেখানে আক্রমণকারী দুটি ডিভাইসের মধ্যে প্রবাহিত সমস্ত ট্রাফিক দেখে বা পরিবর্তন করে। এটি খুবই বিপজ্জনক হতে পারে, বিশেষ করে যদি সেই ট্রাফিকের মধ্যে পাসওয়ার্ড বা অন্যান্য সংবেদনশীল তথ্য থাকে।

আক্রমণকারী ডেটা গন্তব্যে পাঠানোর আগে সেটি ক্যাপচার করতে পারে এবং তারপর সেই ডেটা মডিফাই করে নতুন গন্তব্যে পাঠাতে পারে।


4. ARP Poisoning Tools:

ARP Spoofing বা ARP Poisoning আক্রমণ চালানোর জন্য কিছু জনপ্রিয় টুলস রয়েছে যা সহজেই এই কাজ সম্পন্ন করতে সাহায্য করে। এই টুলসগুলির মধ্যে রয়েছে:

Ettercap:

Ettercap একটি শক্তিশালী এবং জনপ্রিয় MITM টুল যা ARP Poisoning এর জন্য ব্যবহৃত হয়।

এটি গ্রাফিকাল ইউজার ইন্টারফেস (GUI) এবং কমান্ড লাইন মোড (CLI) উভয়েই ব্যবহৃত হতে পারে।

Ettercap আক্রমণকারীকে সহজেই ARP Spoofing করে সিস্টেমে ম্যান-ইন-দ্য-মিডল আক্রমণ চালাতে সাহায্য করে।


Ettercap ব্যবহার:

1. Ettercap ইনস্টল করতে:

sudo apt install ettercap-graphical


2. Ettercap চালাতে:

sudo ettercap -G


3. ARP Spoofing চালানোর জন্য:

Target system সিলেক্ট করুন এবং "Start sniffing" এ ক্লিক করুন।




Cain and Abel:

Cain and Abel Windows-এর জন্য ARP Spoofing এবং পাসওয়ার্ড রিকভারি টুল।

এটি ARP Poisoning, Sniffing, এবং MITM আক্রমণগুলির জন্য ব্যবহার করা হয়।


arpspoof:

arpspoof একটি CLI টুল যা ARP Spoofing আক্রমণ করতে ব্যবহৃত হয়।

এটি ARP Request এবং ARP Reply প্যাকেট ইনজেক্ট করে নির্দিষ্ট IP থেকে MAC অ্যাড্রেস ম্যানিপুলেট করতে সাহায্য করে।


arpspoof ব্যবহার:

1. arpspoof ইনস্টল করতে:

sudo apt install dsniff


2. ARP Spoofing শুরু করতে:

sudo arpspoof -i eth0 -t [Target IP] [Gateway IP]

অথবা

sudo arpspoof -i eth0 -t [Gateway IP] [Target IP]




5. Network Map & Target Selection:

আক্রমণকারীকে সাধারণত নেটওয়ার্কের সঠিক টার্গেট সিলেক্ট করতে হয়। এটি সম্পন্ন করার জন্য আক্রমণকারী সাধারণত Nmap বা Netdiscover ব্যবহার করে নেটওয়ার্কে থাকা ডিভাইসের IP এবং MAC অ্যাড্রেস খুঁজে বের করে।

Nmap: Nmap একটি নেটওয়ার্ক স্ক্যানিং টুল যা আক্রমণকারীকে নেটওয়ার্কের সমস্ত ডিভাইস এবং তাদের IP-MAC অ্যাড্রেস সনাক্ত করতে সাহায্য করে।

Nmap ব্যবহার:

1. Nmap ইনস্টল করতে:

sudo apt install nmap


2. নেটওয়ার্ক স্ক্যান করতে:

sudo nmap -sn 192.168.1.0/24




6. Packet Sniffing & Data Manipulation:

ARP Spoofing-এর মাধ্যমে আক্রমণকারী নেটওয়ার্ক ট্রাফিক স্নিফিং এবং ডেটা ম্যানিপুলেশন করতে পারে। আক্রমণকারী Wireshark বা tcpdump ব্যবহার করে নেটওয়ার্ক ট্রাফিক মনিটর করতে পারে এবং সেসমূহে সংবেদনশীল তথ্য (যেমন পাসওয়ার্ড, ইউজারনেম, ফাইল ট্রান্সফার ইত্যাদি) চুরি করতে পারে।


---

ARP Spoofing থেকে রক্ষা পেতে:

1. স্ট্যাটিক ARP এন্ট্রি:

ARP টেবিলের এন্ট্রি গুলি স্ট্যাটিক করে রাখা যেতে পারে। এতে কোনো ভুয়া ARP Reply আসলে তা গ্রহণ করা হবে না।



2. ARP Spoofing Detection টুলস:

Arpwatch, XArp বা Ettercap এর মতো টুলস ব্যবহার করে ARP Spoofing সনাক্ত করা যায়।



3. প্রোটোকল এনক্রিপশন:

HTTPS বা SSL/TLS এর মতো এনক্রিপ্টেড প্রোটোকল ব্যবহার করা উচিত, যাতে নেটওয়ার্কে ডেটা চুরি বা ম্যানিপুলেশন করা কঠিন হয়ে পড়ে।



4. নেটওয়ার্ক মনিটরিং:

নিয়মিতভাবে নেটওয়ার্ক মনিটরিং এবং অডিট করা উচিত।





---

উপসংহার:

ARP Spoofing একটি শক্তিশালী আক্রমণ, যা নেটওয়ার্ক ট্রাফিক চুরি এবং ম্যানিপুলেশন করার সুযোগ দেয়। এই আক্রমণ প্রতিরোধের জন্য সঠিক সিকিউরিটি প্র্যাকটিস, এনক্রিপশন এবং নিয়মিত মনিটরিং অত্যন্ত গুরুত্বপূর্ণ।
